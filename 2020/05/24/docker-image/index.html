理解Docker的镜像image

2020-05-24 Docker docker 评论

Docker通过容器来打包应用，（环境和代码）

Docker的三大核心概念：

1.镜像（image）

2.容器（container）

3.仓库（repository）

镜像：类似于虚拟机环境，只读的模板，镜像是创建容器的基础

容器：从镜像创建的应用运行实例，盒子，沙箱，互不可见，隔离

仓库：docker集中存放镜像的场所，类似代码仓库

Docker仓库和仓库注册服务器的关系

仓库注册服务器是存放仓库的地方，每个仓库集中存放某一类镜像，通过不同的tag来区分(类似git管理)

注册服务器 --> 创库 --> 各种仓库的不同tag版本
