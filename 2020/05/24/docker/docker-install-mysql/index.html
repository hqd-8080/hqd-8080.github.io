Docker安装MySQL8.0

拉取最新版mysql镜像
docker pull mysql

查看镜像
docker images -a

运行容器
docker run -p 3306:3306 --name mysql -e MYSQL_ROOT_PASSWORD=123456 -d mysql:latest

如果要建立目录映射

检查容器是否正确运行
docker container ls | docker ps -a

进入容器

宿主机链接容器的mysql（远程连接mysql）
我们在创建容器的时候已经将容器的3306端口和主机的3306端口映射到一起，所以我们应该访问：

如果你的容器运行正常，但是无法访问到MySQL，一般有以下几个可能的原因：

caching_sha2_password问题
为了提供更安全的密码加密MySQL8.0的首选默认认证插件是，caching_sha2_password而不是mysql_native_password

或者

防火墙阻拦

开放端口：
systemctl status firewalld
firewall-cmd --zone=public --add-port=3306/tcp -permanent
firewall-cmd --reload

关闭防火墙：
systemctl stop firewalld

需要进入docker本地客户端设置远程访问账号

mysql使用mysql数据库中的user表来管理权限，修改user表就可以修改权限（只有root账号可以修改）
