git flow
  • 1.master主分支，存放稳定的项目版本，这个分支主要是从别的分支合并过来，不会直接在这个分支上提交
  • 2.develop开发分支，当要新增新功能时，所有的feature分支都是从这个分支划分出去的，而feature分支功能完成后也会合并回develop分支
  • 3.hotfix分支，当线上发生紧急问题时，从master分支划分出hotfix进行紧急问题修复，修复完成合并回master，同时合并回develop分支
  • 4.release分支，发布线上分支，develop分支足够成熟可合并到release分支
  • 5.feature分支，新增功能的分支，从develop分支划出，完成后合并回develop分支
